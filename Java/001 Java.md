Java
=======
컴파일언어. 객체지향적.
가상머신(맞습니다! 그 유명한 JVM!..아닌강)을 이용해,
이식성이 뛰어납니다.

예제
--------
```java
public class Hello {
  public static void main (String[] args) {
    System.out.print("반갑수다예, 제주도");
  }
}
```

기본 문법
----------
각 명령행 끝에 세미콜론(;)을 붙입니다.
구조는 
```
소스파일 [
  클래스 [
    메소드1 [
      내용
    ]
    메소드2 [
      내용
    ]
  ]
]
```
형태로(이게뭐야..) 이루어져 있습니다.
- 사용자가 JVM이 없으면 실행할 수 없습니다.
- 각 소스파일에는 하나의 클래스만 선언!
- 모든것이 클래스안으로 들어갑니다
- 모든 애플리케이션에는 최소 클래스 한개이상 필수!
- main메소드는 애플리케이션당 하나만!(없으면 안됨요!)
- 클래스 안에는 메소드가 한개이상 들어갑니다.
- 각 메소드 안에는 우리가 흔히아는 코드( for, if...)들이 들어갑니다.
- 각 클래스, 메소드 내용물은 중괄호{}로 포장합니다.
- 자바에서는 '전역'의 개념이 없습니다. 하지만 public, static 메소드는 전역메소드와 비슷한 효과를 냅니다.
- public, static, final로 지정된 변수도 어디에서든 접근할 수 있는 상수를 만들 수 있습니다.
- 많은 파일들을 '자바 아카이브(.jar 파일)' 하나에 넣을 수 있습니다.
- 자바 아카이브에는 'manifest'라고 부르는 텍스트 파일이 하나 들어가는데 어떤 파일을 실행 시켜야 할지 알려주는 내용이 들어있습니다.
