변수
============

예제
--------
```java
int i;
int k = 3;
char c = 'a';
float PI = 3.14f;
```

문법
--------
자바에서 변수는 '원시변수'와 '객체 레퍼런스', 두가지로 나뉩니다. 
원시변수에는 값을 나타내는 비트가, 객체 레퍼런스에는 객체에 접근하는 방법을 나타내는 비트가 저장됩니다. 

#### 선언
원시변수, 객체 레퍼런스모두 동일하게 적용됩니다. 
1. 변수에는 유형이 있어야 합니다.
2. 변수에는 이름이 있어야 합니다.

#### 리터럴 값?
변수가 아닌 값입니다. 3.14, true, 77 등등...
```java
int i = 32 ;// 32가 리터럴 값
boolean b = false; // false가 리터럴 값
```

#### 변수명
- 알파벳, 밑줄, 달러기호($), 숫자를 조합하여 변수명을 명명할 수 있습니다.
- 알파벳, 밑줄, 달러기호($)중 하나로 시작해야합니다. 숫자로시작하면 안됩니다.
- 예약어는 사용할 수 없습니다(public, int, if ...).

#### 주의사항
* 자바에서 소수점이 들어있는것은 기본적으로 double로 간주하기 때문에 float값으로 쓰고싶다면 맨뒤에 'f'를 붙여야 합니다.

        float f = 3.14f;
* 원시변수에서 작은 크기의 변수값을 큰 크기의 변수로 줄수있지만 그 반대는 안됩니다. 정수 변수, 부동소수점 변수, 부울, 문자끼리도 불가능합니다.

        byte b = 3;
        int a = b; // 가능
        // a = b; 컴파일에러!

원시변수
---------
###### 부울, 문자
- boolean | JVM에 따라 다름 | true, false
- char    | 16비트 | 0~65535

###### 정수
- byte    | 8비트 | -128~127
- short   | 16비트 | -32768~32767
- int     | 32비트 | -2147483648~2147483647
- long    | 64비트 | -짱짱큰 값~짱짱큰 값

###### 부동소수점 소수
- float   | 32비트(바뀔 수 있음)
- double  | 64비트(바뀔 수 있음)

객체 레퍼런스
-------------
객체 레퍼런스 변수에는 객체에 대한 레퍼런스가 들어갑니다. C언어의 포인터라 생각해도 되지만 
무엇이 들어있는지는 확실히 알 수 없습니다. 레퍼런스이기 때문에 
1번 객체를 참고했다가, 2번 객체를 참고하듯이 참고하는 객체를 변경할 수 있습니다.

만약 해당객체를 참고하는 레퍼런스가 하나도 없다면(더이상 그 객체를 사용할수도, 찾을수도 없을때) 해당 객체는
가비지 콜렉터의 대상이 됩니다.

#### 점연산자
레퍼런스 변수에 점 연산자를 사용해서 메소드를 참조하거나 인스턴스 변수를 참조할 수 있습니다.

#### 객체 선언, 생성, 대입 과정
        ... = new Obj(); // 레퍼런스 변수 선언
        Obj tObj = ... // 객체 생성
        Obj tobj = nee Obj(); // 객체와 레퍼런스 연결

#### 배열
배열도 자바에서는 객체입니다. 하지만 원소들은 원시변수입니다. 배열 변수는 배열 객체를 가르키는 레퍼런스 역할을 합니다.
선언은

        int[] arr = new int[len];
        // 타입[] 배열명 = new 타입[길이];
객체 레퍼런스로 배열을 만들든 원시변수로 만들든 해당 배열은 객체입니다.(각 원소는 원시변수또는 객체레퍼런스입니다) 

###### 배열에 있는 인스턴스 변수
- length(배열명.length의 형태로 사용)변수에는 해당 배열의 원소의 갯수를나타내는 값이 들어있습니다.
